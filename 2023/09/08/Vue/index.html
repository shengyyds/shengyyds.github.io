7. 事件处理

vue 绑定事件：

前提：在实例里有 methods

@click="demo" 和@click="demo($event)" 效果一致，但是后者可以传参

事件修饰符

<a href="http://www.baidu.com" @click.prevent="show()">点我学习</a>

常见事件修饰符：

  1. prevent ：阻止默认事件
  2. stop ：阻止事件冒泡
  3. once ：事件只触发一次
  4. capture：使事件在捕获阶段就触发
  5. passive：可以不执行完回调函数就执行，立即执行

拓展：wheel 和 scroll

wheel 是鼠标滚轮，scroll 是绑定滚动事件，无论以什么方式滚动

8. 键盘事件

  1. Vue未提供别名的按键，可以使用按健原始的key值去绑定，但注意要转为kebab-case(短横线命名)
  2. 系统修饰键（用法特殊）:ctrl、alt、shift、meta
    1. 配合keyup使用:按下修饰键的同时，再按下其他键，随后释放其他键，事件才被触发。配合keydown使用:正常触发事件。
    2. 也可以使用keyCode去指定具体的按键(不推荐)
    3. Vue.config.keyCodes.自定义键名=键码，可以去定制按键别名

计算属性的优势是：具有缓存机制，这是 methods 没有的，这样可以减少内存的消耗

  2. 所依赖的数据发生变化时，会被调用
  3. 注意：当重复读取时，读取的是缓存内的数据

get 的作用：当计算属性被读取时，发生调用，返回值就是计算属性的值

set 什么时候调用：

  1. 当参数被修改时调用

10. 监视属性

handle 调用时间：当监视属性发生改变时，下列案例就是 isHot 改变时调用。

监视属性必须存在才能监视

深度监视

Vue 自身是可以监视内部值的改变的，但是提供的watch 默认不可以

如果想监听 numbers 里面 a 的变化

解决：

deep:true, 可以使 handler 监视 numbers 对象内所有变化的东西，包括内部的对象

vue 默认可以监测多级改变，但是 watch 默认不可以检测多级内部改变

watch 和 computed 区别

  1. computed 能完成的功能 ，watch 都可以完成。

  2. watch 能完成的功能，computed 不一定能完成，例如：watch 可以进行异步操作。

    两个重要的小原则:

    1. 所被 Vue 管理的函数，最好写成普通函数，这样this 的指向才是 vm 或组件实例对象。
    2. 所有不被 Vue 所管理的函数（定时器的回调函数、ajax 的回调函数等)，最好写成箭头函数，这样 this 的指向才是 vm 或组件实例对象。

key 的作用原理

场景一：使用index 作为key 的问题：

如果对数据进行了破坏顺序的操作，会触发错误：

场景：原先页面遍历数据库将用户信息拿出放到页面上，当手动台添加一个用户信息，并且打乱原有的顺序进行添加，出现问题。

vue 生成页面元素时，会先生成虚拟dom 然后再转成真实 dom ，若重新开始代码生成时，并且之前生成过一次虚拟dom ，vue 内部会进行一步算法：虚拟 dom 对比算法 ，与原页面相同的部分会进行复用，不同的元素会在新生成的虚拟dom 上转成真实dom ，问题随之产生。

当在代码中未主动标识 key 的值，vue 会自动帮忙补充成 index

解决：使用数据内部的唯一标识进行生成、标记数据

15. Vue.set() 方法

当设计 data 时的数据属性并未设计完整，我们可以使用 Vue.set() 进行添加属性，新添加的属性也还具有 get 和set 方法。

局限性：vue.set() 和 vm.$set() 不能给 vm 或者 vm 的跟数据对象添加属性

16. vue 监测数据改变

检测数组

监测数组内数据改变时，直接修改数组内数据是不会引起页面上数据改变的，只有调用以下七个方法才能够让 vue 承认修改了数据

  1. push 添加
  2. pop 移除
  3. shift 末尾添加
  4. unshift 首位添加
  5. splice 切片
  6. sort 排序
  7. reverse逆序

这七种方法才能够让数据是响应式。

监测对象

通过 setter 实现监测，而且要在 new vue 时就传入要检测的数据

对于后加的属性 vue 默认不做响应式处理

如想要给后添加的属性做响应式，使用：

Vue.set(要修改数据的位置,属性名，属性值)

vm.$set(要修改数据的位置,属性名，属性值)

内置指令

  • v-bind，单向绑定，可简写为 “:”
  • v-model，双向绑定
  • v-for，遍历
  • v-on，绑定监听，简写为@
  • v-if，条件渲染
  • v-else，条件渲染，配合 v-if 一起使用
  • v-show，条件渲染，控制节点是否展示，true，false
  • v-text，插入文本，和插值语法作用一样，会把数据的所有内容当字符串替换掉当前标签的内容。
  • v-html，和 v-text 相像，只不过可以解析字符串

注意： v-html 存在安全行为问题，要在可信的内容上使用它

  • v-cloak，没有值，vue 实例创建完毕后被删除，通过结合 css，可以把未解析的模板隐藏，解决网速慢的时候，页面上全是未解析的模板情况
  • v-once，没有值，在初次动态渲染后，就视为静态内容。不再改变
  • v-pre，没有值，可以让标签跳过编译过程，代码写什么，展示什么，节省资源

21. 生命周期p49

模板->解析->虚拟 dom->真实 dom->挂载->挂载完毕

  1. 又叫生命周期钩子、生命周期函数
  2. 生命周期就是指这些函数在关键的时刻调用特定的函数。
  3. 函数名不能更改，内容自定
  4. this 是 vm

生命周期流程

  1. template 里面只能存在一个根节点，
  2. beforCreat ，无法通过 vm 访问到 data 中的数据、methods 中的方法
  3. created，可以通过 vm 访问到 data 中的数据、 methods 中配置的方法
  4. beforeMount，页面呈现的是未经 Vue 编译的 DOM，所有对 DOM 的操作，最终都不奏效。
  5. mounted，挂载
  6. beforUpdate，在这个时期，页面和数据没同步
  7. updated，数据、页面都是新的
  8. beforeDestroy，销毁前的临终遗言
  9. destroyed，销毁，销毁后 vue 开发者工具看不到任何消息，自定义事件会失效，但是原生 DOM 事件依旧存在

2. VueComponent P59

  1. 组件本质就是一个名为 VueComponent 的构造函数，是 Vue.extend() 创造的
  2. 每次调用 Vue.extend 返回的都是一个全新的 VueComponent
  3. new vue 中的 this 都是 vue，组件中的 this 都是 VueComponent
  4. VueComponent 以下简称 vc ，Vue 的实例对象简称 vm，两者在创建的时候都复用了一部分函数，所以很相似。但是也存在不同点，比如 : vc 中没有 el ，data 也必须使用函数式等。
  5. vc 对象和 vm 对象的关系：
    它们之间是继承关系，vc 对象继承 vm 对象。vc 对象的原型就是 vm 对象。

Vue 和 VueComponent 之间的关系

可以近似看成父类和子类的关系 ，vc 可以调用 vue 身上的函数和方法

实例的原型对象一定来自于缔造者的原型对象
