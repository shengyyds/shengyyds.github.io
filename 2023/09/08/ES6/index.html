7. 箭头函数

允许使用箭头函数

let fn =function(){} 等同于 let fn=(形参) =>{ }

箭头函数的特性

this 是静态的，this 始终指向函数声明时作用域下的 this 的值，call 方法改变不了

不能作为构造实例化对象

不能使用 arguments 变量

简写：省略小括号（当形参有且只有一个时）、省略大括号（当时有一句代码体，此时 return 也要省略）

8. 参数默认值

允许给函数参数赋值初始值

形参初始值具有默认值的参数,一般位置要靠后(潜规则)

可以与解构赋值相结合

9. 引入 rest 函数

用于获取函数的实参，用来代替 arguments

注意：这是浅拷贝

浅拷贝和深拷贝的区分

浅拷贝只复制指向某个对象的指针，内存共享；深拷贝是重新从堆内存中划分一块区域存放数据。

对象赋值是浅拷贝

11. This 和箭头函数

箭头函数提供了一种更加简洁的函数书写方式。基本语法是：

当箭头函数没有参数或者有多个参数，要用 () 括起来

当箭头函数要返回对象的时候，为了区分于代码块，要用 () 将对象包裹起来。

箭头函数能保存函数创建时的 this 值，而不是调用时的值。

箭头函数根本没有自己的 this，导致箭头函数体内部的 this 就是外层函数体的 this，即从作用域链的上一层继承 this。

若普通函数是一个对象的方法，则它的 this 指针指向这个对象
ES6 之前，JavaScript 的 this 对象一直很令人头大，回调函数，经常看到 var self = this 这样的代码，为了将外部 this 传递到回调函数中，那么有了箭头函数，就不需要这样做了，直接使用 this 就行。

不适用的地方

在对象里定义的函数，且该函数内部包括 this 的时候，不适用。

回调函数(callback)里的 this 参数

需要动态this时，也不应该使用动态函数。
